# 02 — 페르소나

## 페르소나 1: 박준호 (PM/기획자)

**배경**
- 직급: 프로젝트 매니저
- 경력: 기획/운영 5년
- 업무: 스프린트 계획, 진행 상황 관리, 우선도 조정

**목표**
- 팀의 진행 상황을 실시간으로 파악
- 병목을 빨리 발견하고 우선순위 조정
- 주간/월간 통계로 성과 보고

**불만**
- 기존 도구(노션, 구글 시트)는 느리고 번거로움
- 실시간 업데이트 안 됨 → 정보 동기화 어려움
- 모바일에서 빠르게 체크 불가

**행동 패턴**
- 매일 아침 9시 스탠드업 전 보드 확인 (3~5분)
- 데스크톱/모바일 왔다갔다 (주로 데스크톱)
- 우선도 높은 카드부터 체크, 기한 임박 카드 주목
- 우선도를 자주 조정 (2~3회/주)
- 주말에도 가끔 확인

**기술 수준**: 높음 (드래그, 단축키, 통계 이해)

---

## 페르소나 2: 이수진 (개발자)

**배경**
- 직급: 시니어 개발자
- 경력: 개발 4년
- 업무: 백엔드 개발, 버그 수정, 코드 리뷰

**목표**
- 자신의 작업 상태를 빠르게 업데이트
- 다음 할 일을 명확히 알고 싶음
- 팀의 진행 상황 파악 (병목 해결)

**불만**
- 개발 중에 자꾸 상태 업데이트 귀찮음
- 모바일에서도 빠르게 카드 옮기고 싶음
- 댓글은 선택사항 (간단하게)

**행동 패턴**
- 매일 아침/점심/저녁 3회 확인 (1~2분)
- 주로 모바일 (자리 자주 옮김, 개발 잠깐 쉴 때)
- 자신의 카드 중심 (다른 카드는 안 봄)
- "In Progress" → "Review"로 자주 이동
- 우선도 높은 카드는 자동으로 눈에 띔

**기술 수준**: 매우 높음 (드래그, 터치 최적화 원함)

## 페르소나 3: 김다현 (디자이너)

**배경**
- 직급: UI/UX 디자이너
- 경력: 디자인 3년 (기술팀에 속하지만 비개발)
- 업무: UI/UX 설계, 리서치, 협업

**목표**
- 팀의 작업 상황을 한눈에 파악
- 내 태스크와 의존성 확인
- 명확한 우선순위와 기한 확인

**불만**
- 도구가 복잡하면 쓰기 싫음
- 개발 용어가 많으면 이해 어려움
- 모바일에서도 쉽게 확인하고 싶음

**행동 패턴**
- 주 2~3회 확인 (월요일, 수요일, 금요일)
- 주로 모바일 (자리 자주 옮김, 회의실 다님)
- 자신의 카드와 관련된 댓글만 봄
- 드래그보다 버튼/메뉴 선호 (터치 환경)
- 기한이 가장 중요 (우선도는 2순위)

**기술 수준**: 중간 (직관적 UI에만 반응)

## 긴장 축 비교

| 축 | 박준호 (PM) | 이수진 (개발자) | 김다현 (디자이너) |
|---|---|---|---|
| **사용 빈도** | 매일 (5분 이상) | 매일 (3회, 1~2분) | 주 2~3회 (3분) |
| **주 디바이스** | 데스크톱 70% + 모바일 30% | 모바일 60% + 데스크톱 40% | 모바일 80% + 데스크톱 20% |
| **우선순위 기준** | 우선도 + 기한 | 기한 > 우선도 | 기한 >> 우선도 |
| **기술 편의성** | 드래그, 단축키 수용 | 드래그 가장 좋음 | 최대한 간단하게 |
| **정보 필요도** | 전체 보드 통계/진행률 | 내 카드 중심 + 병목 감지 | 내 카드만 집중 |
| **입력 빈도** | 높음 (보드 수정 자주) | 중간 (상태 변경만) | 낮음 (기한 수정만) |
| **인터랙션** | 클릭/드래그 모두 | 드래그 선호 | 버튼 선호 |

## 설계 긴장점

### 1. 통계 vs 간단함
- **PM**: 통계와 필터링이 중요 (우측 대시보드)
- **개발자**: 자신의 카드만 보고 싶음 (방해 없이)
- **디자이너**: 너무 복잡하면 안 됨
- **해결**: 기본 UI는 간단, 통계는 별도 탭/패널로 제공

### 2. 드래그 vs 버튼
- **PM & 개발자**: 드래그로 빠르게
- **디자이너**: 버튼 선호 (터치 정확성)
- **해결**: 드래그 + 버튼 양쪽 다 지원

### 3. 정보 밀도
- **PM**: 카드에 우선도, 기한, 담당자 다 보이길 원함
- **개발자**: 핵심 정보만 (기한, 우선도)
- **디자이너**: 너무 많으면 헷갈림 (기한만 필수)
- **해결**: 모바일에서는 아이콘만, 상세는 클릭 후 modal

### 4. 반응형 우선순위
- **개발자**: 모바일 사용이 높음 (드래그 최적화 필요)
- **디자이너**: 모바일 최우선
- **PM**: 데스크톱 효율성도 중요
- **해결**: 모바일 먼저 설계하되 데스크톱은 추가 정보 영역

### 5. 사용 빈도 차이
- **PM**: 매일 여러 번 (스탠드업, 우선도 조정)
- **개발자**: 매일 3회 (짧게)
- **디자이너**: 주 2~3회 (한 번에 짧게)
- **해결**: 빠른 로딩 + 진입장벽 낮춤 (반나절 예산 필수)

## 팀 협업 시나리오

### 시나리오 A: PM이 새 작업 추가
1. PM: 새 카드 추가 (제목만 먼저)
2. 개발자/디자이너: 실시간으로 카드 보임 (Realtime)
3. PM: 우선도 조정 (나중에)

### 시나리오 B: 개발자가 작업 중
1. 개발자: 카드 "In Progress"로 이동 (드래그)
2. PM: 대시보드에서 "진행 중" 개수 증가 인지
3. 디자이너: 필요시 댓글로 리뷰 요청

### 시나리오 C: 기한 임박 감지
1. PM: 기한 12시간 전 알림 받음
2. 개발자: 카드가 빨강으로 강조됨
3. 디자이너: 모바일에서 아이콘으로 인지
