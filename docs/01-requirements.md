# 01 — 요구사항

## 프로젝트 개요

**프로젝트명**: 팀 칸반 보드  
**목적**: 팀이 함께 사용하는 실시간 협업 칸반 보드  
**주 사용자**: 소규모 팀 (3~10명)  
**주 디바이스**: 데스크톱 + 모바일 반응형  
**기술 스택**: React + TypeScript  
**데이터 저장**: Supabase (클라우드 DB + 실시간)  
**시간 예산**: 반나절 (2~3시간)

---

## 기능 요구사항

### 핵심 기능

1. **보드 관리**
   - 하나의 보드 뷰 (스코프: 여러 보드는 v2로)
   - 커스텀 컬럼 추가/삭제/재정렬

2. **카드 관리**
   - 카드 생성/수정/삭제
   - 카드 정보: 제목, 설명, 기한, 우선도(High/Medium/Low)
   - 드래그 & 드롭으로 컬럼 간 이동 및 순서 변경

3. **실시간 협업**
   - Supabase Realtime으로 다른 사용자 변경사항 즉시 반영
   - 구체적 동기화: 카드 생성/수정/삭제, 컬럼 변경, 카드 순서 변경

4. **기본 UI**
   - 칸반 보드 뷰 (컬럼 × 카드)
   - 카드 클릭 시 상세 정보 모달/사이드패널
   - 컬럼 헤더: 이름, 추가 버튼, 삭제 버튼

### 데이터 모델

**columns 테이블**
```
id: UUID (Primary Key)
board_id: UUID (고정 - 하나의 보드만 사용)
name: String (컬럼 이름)
position: Integer (순서)
created_at: Timestamp
```

**cards 테이블**
```
id: UUID (Primary Key)
column_id: UUID (Foreign Key → columns)
title: String (필수)
description: String (선택)
priority: Enum (high, medium, low)
due_date: Date (선택)
position: Integer (컬럼 내 순서)
created_at: Timestamp
updated_at: Timestamp
```

## 비기능 요구사항

### 성능
- 카드 드래그 시 즉시 UI 반영 (낙관적 업데이트)
- Supabase Realtime 지연 ≤ 1초 (일반적 기준)

### 반응형
- 데스크톱: 4개 이상 컬럼 표시
- 태블릿: 2~3개 컬럼, 스크롤 가능
- 모바일: 1개 컬럼, 좌우 스크롤 (또는 컬럼 탭)

### 접근성
- 키보드 네비게이션 기본 (드래그는 마우스/터치)
- 색깔 외 다른 시각 신호 (우선도: 색 + 아이콘)

### 보안
- 인증 제외 (스코프: 향후 추가)
- 데이터 공개 가정 (팀 내부용)

---

## 범위 제외 (Out of Scope)

- 사용자 인증/로그인
- 여러 보드 생성·전환
- 댓글/활동 로그
- 첨부 파일
- 라벨/태그
- 칸반 보드 템플릿
- 검색·필터

---

## 완료 기준 (Definition of Done)

✅ 로컬 또는 배포된 URL에서 앱 실행 가능  
✅ 새 컬럼 추가/삭제 가능  
✅ 새 카드 추가/수정/삭제 가능  
✅ 카드를 다른 컬럼으로 드래그 & 드롭 가능  
✅ 다른 브라우저/디바이스에서 실시간 업데이트 확인 (Supabase Realtime)  
✅ 모바일에서도 동작 확인  
✅ 카드 제목, 설명, 기한, 우선도 입력/수정/표시 가능
