🎨 칸반 보드 UI 시안 비교

팀 협업 칸반 보드 — 3가지 distinctive 시안을 비교하세요

시안 A: Motion-first Kanban
개발자 중심 • 부드러운 드래그 모션

✅ 강점

• 개발자 최적
• 모션 피드백
• 신뢰감 있는 색상
• 드래그 중심

⚠️ 약점

• 통계 보기 어려움
• PM 입장 미흡
• 기한 강조 약함

시안 B: Dashboard-integrated
PM 중심 • 실시간 통계 강조

✅ 강점

• PM 최적
• 통계 한눈에
• 팀 협업 강화
• 친근한 색상

⚠️ 약점

• 모바일 복잡
• 기한 강조 약함
• 반응형 개선 필요

시안 C: Timeline-focused
디자이너 중심 • 기한 타임라인

✅ 강점

• 기한 명확
• 디자이너 최적
• 독특한 구조
• 시간 표현 명확

⚠️ 약점

• 개발자 관점 약함
• 통계 정보 부족
• 타임라인 복잡할 수 있음
